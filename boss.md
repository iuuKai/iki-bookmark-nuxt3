# 简历项目经验（可直接使用）

## 项目名称

`iki-bookmark-nuxt3` 在线书签导航系统（Nuxt3）

## 项目介绍（简历版）

负责开发基于 `Nuxt 3 + Nitro + Pinia + Element Plus` 的在线书签导航系统。项目支持游客浏览公共导航，支持用户通过 `GitHub/Gitee OAuth` 登录后管理个人书签，并将数据持久化到个人仓库（Repo as Database）。我主导实现了网址信息抓取、图标策略、仓库文件读写、权限控制，以及大规模列表渲染性能优化。

## 重点亮点 / 复杂点 / 难点与处理方法

1. OAuth + 仓库数据闭环  
- 难点：登录态、权限、远端仓库读写链路需要协同。  
- 处理：统一 API 封装鉴权与仓库文件操作，Pinia 管理用户态与仓库态，中间件控制访问。

2. 图标抓取策略复杂  
- 难点：不同网站 icon 规则不一致，直接校验会拖慢录入和渲染。  
- 处理：录入阶段只提取 icon 候选（`link rel*="icon"` + `/favicon.ico` 回退），渲染阶段采用轻量降级链路，失败再代理。

3. 大量书签导致渲染卡顿  
- 难点：图片并发加载 + 组件实例过多 + 动画重绘导致交互掉帧。  
- 处理：  
  - 图片加载并发队列控制；  
  - icon loading 使用静态占位替代高频动画；  
  - 分类与卡片按视区和分批渲染；  
  - 菜单动画改为 transform 方案，降低重排开销。

4. 骨架态和数据态时序问题  
- 难点：请求失败或 Promise 悬空会导致页面长期停留骨架态。  
- 处理：请求层保证 resolve/reject 终态，页面 loading 初始策略统一，避免骨架与内容反复切换。

## 优化结果（可量化描述模板）

- 解决页面“骨架态卡死”和多处运行时错误，稳定性明显提升。  
- 批量 icon loading 场景下，侧栏动画卡顿显著降低。  
- 首屏渲染链路从“多次状态闪动”收敛为“骨架 -> 数据一次切换”。  
- 类型检查和样式检查问题收敛，`vue-tsc`/`stylelint` 可通过。

## 技术栈

- 前端：Nuxt 3、Vue 3、TypeScript、Pinia、Element Plus、TailwindCSS
- 服务端：Nitro（Nuxt server API）、Node.js
- 数据与集成：GitHub/Gitee OAuth、Repo API（文件化存储）
- 工程化：Stylelint、TypeScript 类型检查

## 简历投递可用描述（精简版）

独立负责 Nuxt3 书签导航系统的核心开发与性能优化，完成 OAuth 登录、仓库存储、网址元信息抓取、书签管理等全链路能力建设。针对大规模 icon 列表场景，设计并实现图片并发队列、视区分段渲染和低开销动画策略，解决页面骨架态卡住与交互卡顿问题，提升系统稳定性与页面流畅度。
